Быстрая настройка App Engine
Введение

Из этого руководства вы узнаете, как развернуть пример приложения на Java в Google App Engine с помощью плагина App Engine Maven.

Вот из каких этапов состоит это руководство:

    * Создание и запуск приложения "Hello, world!"
      Вы узнаете, как запускать приложение в Google Cloud Shell – прямо в браузере. В конце руководства вы опубликуете свое приложение в Интернете через плагин App Engine Maven.
    * Далее...
      Приложение будет запущено в сервисе. Вы сможете поэкспериментировать с ним или же удалить, а затем создать новое.

    Открываем Google Cloud Platform
    https://console.cloud.google.com/start

    Выбираем проект:
    HelloWorld
    helloworld-2018 (https://console.cloud.google.com/home/dashboard?project=helloworld-2018)

Запустите Google Cloud Shell

    Cloud Shell – это встроенный интерфейс командной строки для консоли. Развертывание приложения будет выполнено с помощью этой программы.

    * Запуск Google Cloud Shell
    Откройте Cloud Shell. Для этого нажмите [>_] на панели навигации вверху страницы.

    * Клонировать образец кода
    Клонируйте код проекта Hello World и перейдите во вновь созданный каталог.

    Для этого откройте Cloud Shell и введите следующую команду:

    Чтобы клонировать пример хранилища:

    TUTORIALDIR=src/helloworld-2018/java_gae_quickstart-2018-01-06-16-11
    git clone https://github.com/GoogleCloudPlatform/appengine-try-java.git $TUTORIALDIR

    Чтобы перейти в каталог с руководствами:
    cd $TUTORIALDIR

Настройка развертывания

Перед вами корневой каталог с примером кода. Здесь хранятся файлы конфигурации приложения.

    * Просмотр кода приложения
    Чтобы просмотреть код приложения, введите следующую команду:
    cat src/main/java/myapp/DemoServlet.java

    Этот сервлет отправляет сообщение Hello, world! в ответ на любой запрос.

    * Просмотр конфигурации
    В Google App Engine используются XML-файлы конфигурации развертывания Java-приложений.

    Чтобы просмотреть файл конфигурации, введите следующую команду:
    cat pom.xml

    Поскольку в приложении helloworld используется инструмент Maven, необходимо создать файл pom.xml, который содержит информацию о проекте и параметры его конфигурации.

Проверьте работу приложения

    * Протестируйте приложение в Cloud Shell
    В Cloud Shell вы можете проверить, правильно ли работает приложение. Это похоже на процесс отладки на локальной машине.

    Для этого введите код:
    mvn appengine:run

    * Откройте приложение в браузере
    Приложение было запущено в Cloud Shell. Чтобы получить к нему доступ, нажмите кнопку "Просмотреть в браузере" для подключения к порту 8080.

    * Завершите предварительный просмотр
    Чтобы удалить экземпляр приложения, нажмите Ctrl+C в окне Cloud Shell.

Создание приложения

    Прежде всего нужно создать приложение App Engine и выбрать регион.

    Введите следующую команду:
    gcloud app create

Завершите подготовку

    * Разверните приложение в Cloud Shell

    Чтобы развернуть приложение с помощью Cloud Shell, введите следующее:
    gcloud config set project helloworld-2018
    mvn appengine:deploy

    * Просмотрите свое приложение
    Поздравляем! Ваше приложение развернуто. Его URL по умолчанию: http://helloworld-2018.appspot.com  Нажмите на эту ссылку, чтобы открыть приложение.

    * Просмотрите состояние приложения
    Просмотреть статус приложения можно на панели управления App Engine.
    Откройте меню в левой части консоли.
    Перейдите в раздел App Engine.

Поздравляем!

    Вы успешно развернули приложение App Engine. Вот что можно сделать дальше:

    1. Скачайте Google Cloud SDK и запустите его на локальном компьютере
    После загрузки нужно будет извлечь файл и инициализировать SDK.

    2. Создайте ещё одно приложение
